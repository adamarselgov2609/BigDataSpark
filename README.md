# BigDataSpark — Лабораторная работа №2

ETL-пайплайн на **Apache Spark** (PySpark): загрузка сырых данных в **PostgreSQL**, построение модели **«звезда»** в той же БД и выгрузка шести аналитических витрин в **ClickHouse** (обязательная NoSQL по заданию).

## Что делает проект

1. **Источник** — 10 CSV-файлов (`MOCK_DATA.csv` … `MOCK_DATA (9).csv`, по 1000 строк), импортируются в таблицу `mock_data` при старте PostgreSQL.
2. **Spark Job 1** (`spark_etl_star.py`) — читает `mock_data` по JDBC и строит в PostgreSQL измерения и факт: клиенты, продукты, магазины, поставщики, время, продажи.
3. **Spark Job 2** (`spark_clickhouse_reports.py`) — читает звезду из PostgreSQL, агрегирует метрики и записывает **6 таблиц-витрин** в базу `reports_db` в ClickHouse; для типовых срезов создаются **SQL-представления** (views).


## Технологии

| Компонент | Назначение |
|-----------|------------|
| **Docker / Docker Compose** | Оркестрация сервисов |
| **PostgreSQL 15** | Хранение сырых данных и схемы «звезда» |
| **Apache Spark 3.5** (образ `apache/spark`, PySpark) | ETL и расчёт витрин |
| **ClickHouse** | Колоночное хранилище для отчётов (HTTP 8123, native 9000) |
| **JDBC** | `postgresql` + `clickhouse-jdbc` для чтения/записи из Spark |

## Запуск

**Требования:** установленные Docker и Docker Compose; CSV лежат в каталоге `исходные данные/` (как в репозитории).

```bash
docker compose up --build
```

После старта сервисов автоматически выполняются оба `spark-submit` (сначала звезда, затем витрины в ClickHouse).

Повторный прогон только Spark-джобов:

```bash
docker compose up spark-etl-job
```

**Порты:** PostgreSQL — `5433` → 5432 в контейнере; ClickHouse — `8123`, `9000`.

## Витрины в ClickHouse (`reports_db`)

Каждая витрина — отдельная таблица; детальные срезы — через представления.

| Витрина | Таблица | Содержание (кратко) |
|---------|---------|---------------------|
| **Продажи по продуктам** | `report_products` | Выручка, объём, рейтинг, отзывы по продукту/категории; топ-10, выручка по категориям — во views `v1`–`v3` |
| **Продажи по клиентам** | `report_customers` | Траты, заказы, средний чек; топ-10, страны, средний чек — `v4`–`v6` |
| **Продажи по времени** | `report_time` | Месячные/годовые тренды, сравнение по годам, средний чек по месяцам — `v7`–`v9` |
| **Продажи по магазинам** | `report_stores` | Выручка и средний чек по магазину; топ-5, география — `v10`–`v12` |
| **Продажи по поставщикам** | `report_suppliers` | Выручка, средняя цена товара, распределение по странам поставщика — `v13`–`v15` |
| **Качество продукции** | `report_quality` | Рейтинги (макс./мин.), связь рейтинга с объёмом продаж, отзывы — `v16`, `v16b`, `v17`, `v18` |
